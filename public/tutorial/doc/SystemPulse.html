systempulse
Path: CARNOT/basics

Purpose
Then it will generated a discrete pulse of 0 and 1, but this pulse depends on the system time, not the simulation time.

Description
This function is similar to the SIMULINK pulse function, but it depends on the system time, not on the simulation time. If the date flag is set, the function starts pulsing only when the system date reaches the specified date. Otherwise the output is 0. If the enable flag is set, the function waits, until the enabling pulse will rises once from 0 to 1. Afterwards the pulsing is started, before pulsing the output is 0.

Inputs:

  • enable pulse

Outputs:

  • 0 or 1

Parameters:

  • date [month day hour minute second]
  • period [s]
  • pulsewidth[s]
  • flag, if to wait for start date
